# Тестовое задание: Веб-сервис для управления движением денежных средств (ДДС)

## Описание

**ДДС (движение денежных средств)** — это процесс учета, управления и анализа поступлений и списаний денежных средств компании или частного лица.

Необходимо разработать веб-приложение, которое позволяет пользователям создавать, редактировать, удалять и просматривать записи о движении денежных средств.

Приложение должно иметь удобный и интуитивно понятный интерфейс, а также обеспечивать соблюдение логических зависимостей между сущностями.

---

# Возможности приложения

## Создание записи о движении денежных средств

### Поля записи

- **Дата создания записи**
  - Заполняется автоматически.
  - Может быть изменена вручную.
  - Пример: `01.01.2025`.

- **Статус**
  - Возможные значения:
    - Бизнес
    - Личное
    - Налог
  - Список должен поддерживать расширение.

- **Тип**
  - Возможные значения:
    - Пополнение
    - Списание
  - Список должен поддерживать расширение.

- **Категория** и **подкатегория**
  - Пример:
    - Категория **«Инфраструктура»**
      - VPS
      - Proxy
    - Категория **«Маркетинг»**
      - Farpost
      - Avito
  - Список категорий и подкатегорий должен поддерживать расширение.

- **Сумма**
  - Размер денежных средств в рублях.
  - Пример: `1 000 ₽`.

- **Комментарий**
  - Произвольный текст.
  - Поле необязательно для заполнения.

# Логические зависимости

Необходимо соблюдать следующие зависимости:

- Подкатегории принадлежат категориям.
- Категории принадлежат определенному типу.

### Пример

Тип **«Списание»**

- Категория **«Маркетинг»**
  - Farpost
  - Avito

- Пользователь **не может выбрать подкатегорию**, если она не принадлежит выбранной категории.
- Пользователь **не может выбрать категорию**, если она не относится к выбранному типу.
